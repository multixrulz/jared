Python in the Browser – the verdict

My search is over, and the winner is … Brython. It’s really cool, works properly, is easy to use, and is building in the Python standard library … what’s not to like? OK, apart from being slower than pure javascript.

But in the wider context, nobody wins this contest. My goal is to write software once and have it run everywhere – desktop, browser, mobile – and mobile is the issue. My phone, which is not too hot but probably better than average for my target demographic, really doesn’t run any of the examples well. It’s got issues with execution speed, touch dragging, and sometimes javascript implementation. Basically, I’m not going to get the results I want with browser apps, which is a great pity.
